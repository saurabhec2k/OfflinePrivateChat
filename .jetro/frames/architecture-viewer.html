🏗️ Antigravity - Complete System Architecture

1. Project Overview

Antigravity is a full-stack chat application that enables users to interact with local LLMs (via Ollama) while uploading and processing documents/images. The system emphasizes modularity, scalability, and clean separation of concerns.

Tech Stack

  • Frontend: React 18 + Vite (development) + React Router + Axios
  • Backend: Express.js (Node.js) with middleware-driven architecture
  • LLM Integration: Ollama (local, no API keys required)
  • Document Processing: pdf-parse, mammoth (DOCX), sharp (images)
  • File Handling: Multer for multipart uploads
  • Development: Concurrently (parallel dev servers)

3. Key Layers

🌐 Frontend Layer

  • Components: Reusable React components (ChatFeed, ChatInput, Sidebar)
  • Services: API client layer (axios interceptors)
  • Hooks: Custom React hooks (useChat, useDocuments)
  • Utils: Formatting, validation, constants

🔧 Backend Layer

  • Routes: API endpoints (/chat, /documents, /models, /health)
  • Services: Business logic (ChatService, DocumentService, OllamaService)
  • Middleware: CORS, logging, validation, error handling
  • Utils: Config, logger, validators, error handlers

🦙 LLM Integration

  • OllamaService: Handles all LLM operations
  • Model Management: Pull, list, check models
  • Generation: Stream responses with timeout handling

4. API Endpoints

Method Endpoint Purpose
POST /api/chat Send message to LLM
GET /api/chat/conversations List all conversations
GET /api/chat/:id Get messages in conversation
DELETE /api/chat/:id Delete conversation
POST /api/documents/upload Upload file
GET /api/documents List documents
GET /api/models List available models
GET /api/health System health check

5. Data Flow

💬 Chat Message Flow

User Input → ChatService → Ollama → Response → UI Update

📄 Document Upload Flow

File Select → Upload → Parsing → Content Storage → List Update

6. State Management

Frontend State (Context + Hooks):
  • ChatContext - conversations, messages, loading state
  • DocumentContext - documents, upload status
  • ModelContext - available models, active model

7. Error Handling

  • Input Validation: 400 Bad Request
  • Not Found: 404 Resource not found
  • Service Errors: 500 Internal Server Error
  • Ollama Issues: 503 Service Unavailable

8. Security Features

  • ✅ CORS configuration for origin validation
  • ✅ Input validation on server-side
  • ✅ File type and size validation
  • ✅ Secure error messages (no stack traces)
  • ✅ Environment-based configuration

9. Scalability Path

Phase 1 (Current): Single server, in-memory storage
Phase 2: Database integration, Redis caching
Phase 3: Multi-server, load balancing, CDN

10. Performance Optimizations

  • Code splitting in React
  • Request timeout handling
  • Async document processing
  • Response caching headers
  • Compression enabled
